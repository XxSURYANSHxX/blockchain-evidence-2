Data Protection & Blockchain Transparency

Understanding what data is stored where and how it's protected

Important: Blockchain Transparency

Evidence hashes are stored on blockchain for transparency. Full evidence files remain private and secure.

1. Blockchain vs Traditional Storage

On Blockchain (Transparent)

What's Stored:

  • File hashes (SHA-256)
  • Wallet addresses
  • Timestamps
  • Transaction records

Why It's Safe:

  • No personal data
  • No file contents
  • Only cryptographic proofs
  • Enables public verification

Secure Database (Private)

What's Stored:

  • Actual evidence files
  • Personal information
  • Case details
  • User profiles

Protection Measures:

  • End-to-end encryption
  • Role-based access
  • Regular security audits
  • Compliance monitoring

2. Data Flow & Protection

Evidence Upload

Files encrypted and stored in secure database

→

Hash Generation

Cryptographic hash created from file content

→

Blockchain Record

Only hash and metadata stored on blockchain

3. Privacy Protection Measures

Encryption

  • AES-256 encryption at rest
  • TLS 1.3 for data in transit
  • End-to-end encryption

Access Control

  • Role-based permissions
  • Multi-factor authentication
  • Audit logging

Data Minimization

  • Only necessary data collected
  • Pseudonymization where possible
  • Regular data cleanup

Compliance

  • GDPR compliant
  • DPDP Act adherence
  • Regular audits

4. What Can Be Seen Publicly

Safe for Public View

  • File Hashes: Cryptographic fingerprints (cannot reverse to original file)
  • Wallet Addresses: Blockchain addresses (pseudonymous)
  • Timestamps: When evidence was submitted
  • Transaction IDs: Blockchain transaction references

Never Public

  • File Contents: Actual evidence files
  • Personal Names: Real names and identities
  • Case Details: Sensitive case information
  • Private Keys: Wallet private keys

5. Legal Compliance Framework

GDPR (EU)

  • Right to be forgotten (where legally permissible)
  • Data portability
  • Consent management
  • Privacy by design

DPDP Act (India)

  • Data localization requirements
  • Consent mechanisms
  • Data breach notifications
  • Cross-border transfer rules

Legal Evidence

  • Chain of custody preservation
  • Evidence integrity maintenance
  • Court admissibility standards
  • Retention period compliance

6. Data Subject Rights

Right to Access

Request information about your personal data processing

Right to Rectification

Correct inaccurate or incomplete personal data

Right to Erasure

Request deletion where legally permissible

Right to Portability

Export your data in machine-readable format

Important: Some rights may be limited due to legal evidence preservation requirements and blockchain immutability.
